# 训练主线一：训练基座

## 1. 文档定位

本文描述 `cfie_training` 的训练基座主链。

这里的“训练基座”强调的是：

- bucket-local 的训练执行闭环；
- dataset-backed 的批次规划；
- checkpoint / resume 的可恢复性；
- 资源受限客户端场景下的训练可运行性。

## 2. 当前主链结论

当前训练基座已经不再保留 synthetic batch 作为默认入口。

训练命令当前要求：

- 显式提供 `--dataset`；或
- 从一个 dataset-backed session checkpoint 继续 `resume`。

这意味着训练主线一当前的事实口径是：

1. 批次来源必须可追溯到真实文本数据。
2. planner checkpoint 只恢复 `tokenized_dataset` 规划器。
3. `train` 不再偷偷回退到 synthetic planner。

## 3. 已落地能力

- `txt/jsonl + tokenizer` 的 dataset-backed batch planner。
- `train` / `simulate` / `estimate-startup` / `validate` 命令入口。
- bucket-local `forward -> backward -> update -> release` 闭环。
- session checkpoint 与 resume。
- transport / buffer / runtime snapshot 的持久化。

## 4. 当前关键入口

- `cfie_training/cli/main.py`
  - `validate`
  - `simulate`
  - `estimate-startup`
  - `train`
- `cfie_training/runtime/project.py`
- `cfie_training/runtime/session.py`
- `cfie_training/runtime/data.py`

## 5. 当前验收要点

训练基座当前重点不是“能否用 synthetic 跑起来”，而是：

- dataset-backed 训练是否稳定；
- checkpoint / resume 是否仍保持一致；
- planner 状态是否只来自真实 dataset 路径。

## 6. 当前不再保留的主线路径

- `plan`
- `qwen35-plan`
- synthetic batch planner 默认回退

这些路径如果还需要查看，只能作为历史背景，不再代表当前训练主线。
