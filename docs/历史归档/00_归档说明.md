# 归档说明

## 1. 本次整理做了什么

- 合并了原先散落在 `docs/` 顶层的大量会话交接文档。
- 清理了原始 `.txt` 构建日志、pytest 日志、smoke 日志。
- 清理了过程型 `.md` 日志，只保留长期有效的路线、架构、台账与验证基线。
- 保留了一份长期有价值的单卡推理基线文档：`02_Windows单卡推理验证基线.md`。

## 2. 为什么要这样整理

之前 `docs/` 顶层混入了大量：

- 会话交接文档
- 构建日志
- pytest 输出
- smoke 输出
- 算子盘点与阶段性过程记录

这类文件对“阶段推进”有价值，但对“长期阅读、AI 续做、人工查阅”干扰很大。

## 3. 现在的处理原则

- **长期有效**：保留在 `路线文档/` 与 `架构图文档/`。
- **阶段结论**：压缩进 `01_Windows适配阶段交接汇总.md`。
- **验证基线**：保留 `02_Windows单卡推理验证基线.md`。
- **原始日志**：不再堆在 `docs/` 顶层；后续需要时以运行命令重新生成，或查看仓库中的其他日志目录。
